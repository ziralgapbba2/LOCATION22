# 📦 재고 위치 조회 PWA 설치 가이드

## 🎯 PWA란?
Progressive Web App - 웹사이트를 앱처럼 사용할 수 있게 만드는 기술입니다.
APK 파일 없이도 홈 화면에 설치하여 앱처럼 사용 가능합니다!

## 📂 파일 구성
- `inventory-pwa.html` - 메인 앱 파일
- `manifest.json` - 앱 설정 파일
- `service-worker.js` - 오프라인 작동 파일

## 🚀 배포 방법

### 방법 1: GitHub Pages (추천)
1. GitHub 저장소 생성
2. 3개 파일 모두 업로드
3. Settings → Pages → 활성화
4. 생성된 URL 접속

### 방법 2: 로컬 서버
1. Python이 설치된 경우:
   ```bash
   python -m http.server 8000
   ```
2. 브라우저에서 `http://localhost:8000/inventory-pwa.html` 접속

### 방법 3: 웹 호스팅
- Netlify, Vercel 등에 3개 파일 업로드

## 📱 모바일 설치 방법

### Android (Chrome)
1. 모바일에서 앱 URL 접속
2. 주소창 오른쪽 메뉴(⋮) 클릭
3. "홈 화면에 추가" 선택
4. "추가" 버튼 클릭
5. ✅ 홈 화면에 앱 아이콘 생성!

### iPhone (Safari)
1. Safari에서 앱 URL 접속
2. 하단 공유 버튼(□↑) 클릭
3. "홈 화면에 추가" 선택
4. "추가" 버튼 클릭
5. ✅ 홈 화면에 앱 아이콘 생성!

## 💻 PC 설치 방법

### Chrome/Edge
1. PC에서 앱 URL 접속
2. 주소창 오른쪽 설치 아이콘(⊕) 클릭
3. "설치" 버튼 클릭
4. ✅ 독립 창으로 앱 실행!

## ✨ PWA 장점

✅ **설치 간편** - APK 파일 불필요
✅ **자동 업데이트** - 서버 파일 수정시 자동 반영
✅ **오프라인 작동** - 인터넷 없어도 사용 가능
✅ **용량 적음** - 앱스토어 앱보다 가벼움
✅ **크로스 플랫폼** - Android, iOS, PC 모두 지원

## 🔧 주의사항

1. **HTTPS 필요**: GitHub Pages는 자동으로 HTTPS 제공
2. **3개 파일 모두 필요**: 같은 폴더에 있어야 함
3. **브라우저 지원**: 최신 Chrome, Safari, Edge 권장

## 📞 문제 해결

### "홈 화면에 추가" 버튼이 안 보여요
- HTTPS로 접속했는지 확인
- manifest.json 파일이 있는지 확인
- 브라우저 업데이트 확인

### 오프라인에서 작동 안 해요
- 한 번 이상 온라인에서 실행 필요
- Service Worker가 등록되었는지 확인

### 아이콘이 제대로 안 나와요
- 캐시 삭제 후 재설치
- 브라우저 강력 새로고침 (Ctrl+Shift+R)

## 🎉 사용 시작!

1. GitHub Pages에 배포
2. URL을 팀원들에게 공유
3. 각자 홈 화면에 추가
4. 앱처럼 사용!

데이터 업데이트가 필요하면 서버의 파일만 수정하면 
모든 사용자에게 자동으로 반영됩니다! 🚀
